# План типизации LR_TS

## Порядок выполнения по файлам

### 1. Инфраструктура
- Установить: `typescript`, `ts-loader`, `@types/node`
- Создать `tsconfig.json` (strict, noImplicitAny, noUncheckedIndexedAccess)
- В webpack: `resolve.extensions` добавить `.ts`, `.tsx`; в `rules` — ts-loader для `.ts`
- Точку входа сменить на `./src/index.ts`
- ESLint: parser `@typescript-eslint/parser`, plugin `@typescript-eslint`, ext `.ts`

### 2. Типы данных (`src/types/`)
- **api.ts**: `LoaderOptions`, `RequestEndpoint`, `RequestParams`, `HttpMethod`
- **news.ts**: `NewsArticle` (author, title, description, url, urlToImage, publishedAt, content, source), `NewsSourceRef` (id, name), `NewsResponse` (status, totalResults, articles)
- **sources.ts**: `Source` (id, name) — при необходимости связать с NewsSourceRef через утилиты (Pick/Omit)
- **common.ts**: тип для callback'ов API, например `Callback<T> = (data: T) => void`

Утилиты: `ArticleCardData = Pick<NewsArticle, 'title' | 'url' | 'description' | 'urlToImage' | 'publishedAt' | 'author' | 'source'>` и т.п. где есть пересечения.

### 3. `src/components/controller/loader.js` → `loader.ts`
- Класс `Loader`: типизировать конструктор `(baseLink: string, options: LoaderOptions)`
- Поля: `baseLink: string`, `options: LoaderOptions`
- `getResp({ endpoint, options }: RequestParams, callback?: Callback<unknown>): void`
- `errorHandler(res: Response): Response`
- `makeUrl(options: Record<string, string>, endpoint: RequestEndpoint): string`
- `load(method: HttpMethod, endpoint: RequestEndpoint, callback: Callback<unknown>, options?: Record<string, string>): void`
- Дженерик: можно ввести `getResp<T>(..., callback?: Callback<T>)` и `load<T>(..., callback?: Callback<T>)`, чтобы callback получал типизированные данные.

### 4. `src/components/controller/appLoader.js` → `appLoader.ts`
- Конструктор вызывает `super(process.env.API_URL, { apiKey: process.env.API_KEY })` — типы из Loader; для env использовать `process.env as { API_URL?: string; API_KEY?: string }` или объявить типы в `env.d.ts`.

### 5. `src/components/controller/controller.js` → `controller.ts`
- `getSources(callback: Callback<SourcesResponse>): void`
- `getNews(e: MouseEvent, callback: Callback<NewsResponse>): void`
- Внутри: `target` и `newsContainer` — типизировать как `EventTarget | null` и приводить к `HTMLElement` после проверок; использовать `e.currentTarget as HTMLElement` с проверкой.

### 6. `src/components/app/app.js` → `app.ts`
- Поля: `controller: AppController`, `view: AppView`
- `start(): void`
- Типизировать `querySelector('.sources')` как `HTMLElement | null` и проверять перед `addEventListener`.

### 7. `src/components/view/appView.js` → `appView.ts`
- Поля: `news: News`, `sources: Sources`
- `drawNews(data: NewsResponse | undefined): void`
- `drawSources(data: SourcesResponse | undefined): void`
- Внутри: `values` — типизировать как `NewsArticle[]` и `Source[]`.

### 8. `src/components/view/news/news.js` → `news.ts`
- `draw(data: NewsArticle[]): void`
- Все переменные: `news`, `fragment`, `newsItemTemp`, `newsClone`, селекторы — типизировать (DocumentFragment, HTMLTemplateElement, NodeListOf, HTMLElement).
- Элементы из `querySelector` — проверять на null или использовать non-null assertion только там, где структура HTML гарантирована.

### 9. `src/components/view/sources/sources.js` → `sources.ts`
- `draw(data: Source[]): void`
- Аналогично: fragment, sourceItemTemp, sourceClone и селекторы — полная типизация DOM.

### 10. `src/index.js` → `index.ts`
- `const app: App = new App();` и вызов `app.start()`.

### 11. Проверки
- `npx tsc --noEmit`
- `npm run lint`
- `npm run build`
- Убедиться: нет `any`, нет висячих типов, нет `as any as`.

---

## Где использовать утилиты типов
- `Source` и ссылка на источник в статье: если поле `source` в `NewsArticle` совпадает по форме с частью `Source`, использовать `NewsArticle['source']` или общий тип.
- Параметры запроса (options): `Record<string, string>`.
- Callback: общий тип `Callback<T>` в common.

## Где дженерики
- `Loader.load<T>(..., callback: Callback<T>)` и `getResp<T>(..., callback?: Callback<T>)` — чтобы AppController мог передавать типизированные callback'и (NewsResponse, SourcesResponse).

- Типизированы все переменные, параметры и возвращаемые значения; дженерики в `getResp`/`load`; утилита `ArticleCardData = Pick<...>` в типах новостей.
- Проверки: `npx tsc --noEmit` и `npm run build` проходят. В коде нет `any`.

## Перед сдачей
1. Выполни `npm install` в корне LR_TS (для typescript, ts-loader, @types/node).
2. (Опционально) Для линтинга TS: `npm i -D @typescript-eslint/parser @typescript-eslint/eslint-plugin`, затем в `.eslintrc.json` указать `parser: "@typescript-eslint/parser"` и плагин.
3. Финально: `npx tsc --noEmit`, `npm run build`, убедись что нет `any` и висячих типов.
